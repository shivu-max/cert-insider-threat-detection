# 🛡️ CERT Insider Threat Detection System

## 📌 Overview
The **CERT Insider Threat Detection System** is a machine learning project designed to detect suspicious employee activities inside an organization using behavioral analytics and anomaly detection techniques.

Insider threats occur when trusted employees misuse their access privileges to compromise sensitive organizational resources such as confidential data, internal systems, or intellectual property.

This project analyzes **user activity logs from the CERT Insider Threat Dataset (r4.2)** and identifies abnormal patterns that may indicate potential insider threats. By applying machine learning models and data analysis techniques, the system helps security teams detect risky behavior early and take preventive actions.

---

# 🎯 Problem Statement
Organizations face increasing cybersecurity risks from **insider threats**, where employees, contractors, or trusted users misuse their access privileges intentionally or unintentionally.

Traditional security systems often fail to detect insider threats because malicious actions often resemble normal user behavior.

This project aims to:

- 🔍 Analyze large-scale employee activity logs  
- 🚨 Detect anomalous behavioral patterns  
- 🤖 Identify potential insider threats using machine learning  
- 📊 Provide processed outputs for further security analysis  

---

# 📂 Dataset

This project uses the **CERT Insider Threat Dataset (Release r4.2)** developed by the **Carnegie Mellon University CERT Division**.

The dataset simulates activity logs from an enterprise environment including multiple types of employee activities.

### Dataset Download

Official Dataset Link:

https://kilthub.cmu.edu/articles/dataset/Insider_Threat_Test_Dataset/12841247

The dataset includes multiple log files such as:

- logon.csv → user login and logout activity  
- email.csv → email communication records  
- device.csv → USB and external device usage  
- file.csv → file access operations  
- http.csv → web browsing activity  

These logs simulate **17+ months of organizational activity across thousands of users**, including both normal and malicious insider behavior.

# 📊 Processed Outputs

After running the machine learning pipeline, the system generates processed results stored inside the **processed_outputs** directory.

These outputs help security analysts identify suspicious behavior patterns.

### 📄 Generated CSV Files

**suspicious_users.csv**

Contains users flagged as potential insider threats based on abnormal behavioral patterns.

Fields include:

- user_id  
- anomaly_score  
- threat_level  
- suspicious_activity  

---

**anomaly_scores.csv**

Contains anomaly detection scores generated by the machine learning model.

Fields include:

- user_id  
- anomaly_score  
- activity_frequency  
- deviation_score  

---

**processed_activity_logs.csv**

Cleaned and transformed activity logs used for model training and analysis.

Fields include:

- user_id  
- login_time  
- file_access_count  
- device_usage  
- web_activity  

---

**threat_predictions.csv**

Final predictions produced by the trained machine learning model.

Fields include:

- user_id  
- predicted_label  
- probability_score  
- threat_classification
